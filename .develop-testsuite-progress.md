## Summary

The UC-12348 Robot Framework test suite has been developed and verified.

### Changes Made

1. **Phone discovery** (`robot/resources/voice.resource`): Location-based discovery - no hardcoded device names. Phones are classified by network location (LAN/WAN) using device metadata (`options` field) or naming convention. Tests use `Setup Phones For Scenario caller_location=LAN callee_location=WAN` to assign phones by type and quantity. Works across different testbeds (2, 3, or more phones).

2. **Missing scenarios** (`robot/tests/user_makes_one_way_call.robot`):
   - UC-12348-Main-V2: LAN Phone Calls WAN Phone 2 (NAT Outbound)
   - UC-12348-Main-V5: WAN Phone 2 Calls LAN Phone (NAT Inbound)
   - UC-12348-Main-V6: WAN Phone 2 Calls WAN Phone (Direct WAN)
   - UC-12348-15a: RTP Media Fails To Establish

3. **Get Device Property** (`robot/libraries/boardfarm_keywords.py`): Added keyword to read device attributes (e.g. `number` from SIPPhone).

4. **Unit tests for voice keywords** (`robot/utest/test_voice_keywords.py`): Added 13 unit tests for `discover_phones_by_location`, `Discover Phones By Location` keyword, and `Assign Phones For Scenario` keyword. Covers both pass (statement true) and fail (statement false) outcomes per develop-testsuite skill.
